算法

复杂度计算

  • 常对幂指阶
    $O(1) < O(log_2^n) < O(n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n) $

  • 加法规则
    $ O(f(n)) + O(g(n)) = O(max(f(n), g(n))) $

  • 乘法规则
    $ O(f(n)) \times O(g(n)) = O((f(n) \times g(n))) $

时间复杂度

  • 计算

    1. 找到一个基本操作（最深层循环）
    2. 分析该基本操作的执行次数 x 与问题规模 n 的关系 n=f(x)
    3. x 的数量级 O(x) 就是算法时间复杂度 T(n) $ T(n) = O(n) + O(n^2) = O(n^2) $

  • 结论

    1. 顺序执行的代码只会影响常数项，可忽略
    2. 只需挑选循环中的一个基本操作分析它执行次数与 n 的关系
    3. 如果有多层嵌套循环，只关注最深层循环次数

空间复杂度

  • 普通程序计算
    1. 找到所占空间大小与问题规模的变量
    2. 分析所占空间 x 与问题规模 n 的关系 x=f(n)
    3. x 的数量级 O(x) 就是算法空间复杂度 S(n) $ S(n) = O(n^2) + O(n) + O(1) = O(n^2) $

  • 递归程序计算

    1. 找到递归调用的深度 x 与问题规模 n 的关系 x=f(n)
    2. x 的数量级 O(x) 就是算法空间复杂度 S(n) ( digui_2 中因函数中的定义存在额外开销 ) $递归1: S(n) = O(n) $
      $递归2: S(n) = O(n^2) $

  • 原地工作
    算法所需内存空间为常量 O(1)
